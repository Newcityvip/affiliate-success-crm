# Database

Affiliate Success CRM uses Google Sheets as the database for the early product phases. The schema below is finalized for the current foundation and must not be changed without an explicit schema change process.

## Finalized Tabs

| Tab | Purpose |
| --- | --- |
| `Affiliates` | Stores affiliate profile records and core relationship fields. |
| `Staff_List` | Stores staff identities and role/access reference fields. |
| `Brand_List` | Stores brand reference fields used by CRM workflows. |
| `Weekly_Contacts` | Stores weekly affiliate contact planning and tracking fields. |
| `Monthly_Performance` | Stores monthly affiliate performance tracking fields. |
| `Task_Log` | Stores internal task tracking fields. |
| `Issue_Log` | Stores affiliate or operational issue tracking fields. |
| `Dashboard_Config` | Stores dashboard configuration fields. |
| `Interaction_Log` | Stores affiliate interaction history fields. |
| `KPI_Config` | Stores KPI configuration fields. |
| `Activity_Log` | Stores audit and activity log fields. |
| `Followup_Queue` | Stores follow-up queue fields. |

No additional tabs are introduced in Sprint 0.
